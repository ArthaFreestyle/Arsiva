# 🔐 Feature: Role-Based Content Visibility — Split `super_admin` & `guru` Content Management

## 📋 Summary

Currently, the **"Get All" endpoints** for content (Puzzle, Cerita Interaktif, Kuis) only return **published content** (`is_published = true`) and do not differentiate between roles. Both `guru` and `super_admin` see the exact same data.

This issue implements **role-based content visibility** so that:
- **`guru`** → Can only see and manage content **they created** (filtered by `created_by = user_id`)
- **`super_admin`** → Can see and manage **ALL content** created by every user (no filter)

This applies to the **management/write endpoints** (under the `guruAdmin` middleware group). The **read-only endpoints** (under `allAuth`) remain unchanged — they continue to show only published content to all authenticated users (including `member`).

---

## 🗄️ Database Context

All three content tables share the same ownership pattern via the `created_by` column:

### Table: `puzzles`
```sql

    created_at TIMESTAMP(0) NOT NULL DEFAULT CURRENT_TIMESTAMP,
    is_published BOOLEAN NOT NULL DEFAULT false,
    CONSTRAINT puzzles_pkey PRIMARY KEY (puzzle_id)
);
```

> All three tables have `created_by INTEGER NULL` which references `users(user_id)`. This is the column we use to filter content by creator.

### JWT Claims Context
```go
type Claims struct {
    UserId   string `json:"user_id"`
    Email    string `json:"email"`
    Username string `json:"username"`
    Role     string `json:"role"`     // "super_admin", "guru", or "member"
    jwt.RegisteredClaims
}
```

The `role` and `userId` are available in every authenticated handler via:
```go

```

## 🏗️ Architecture Context

### Current Flow (No Role Distinction)

```
Controller.GetAllPuzzle()
    → extracts: page, size, search
    → calls: UseCase.GetAllPuzzle(ctx, page, size, search)
        → calls: Repository.FindAll(ctx, page, size, search)
            → SQL: WHERE is_published = true AND judul ILIKE $1
```

The same pattern repeats for Cerita and Kuis. **No `userId` or `role` is passed** to the usecase/repository `GetAll` methods, so filtering by creator is impossible.

### Target Flow (Role-Aware)

```
Controller.GetAllPuzzle()
    → extracts: page, size, search, userId, role
    → calls: UseCase.GetAllPuzzle(ctx, page, size, search, userId, role)
        → calls: Repository.FindAll(ctx, page, size, search, userId, role)
            → if role == "super_admin":
                  SQL: WHERE judul ILIKE $1           (all content, published or not)
              else (guru):
                  SQL: WHERE created_by = $X AND judul ILIKE $1    (only own content)
```

##### 1.1.2 Implement `FindAllManage`

This method dynamically builds the SQL `WHERE` clause based on the role:

```go
func (r *puzzleRepositoryImpl) FindAllManage(ctx context.Context, page int, size int, search string, userId string, role string) ([]*entity.Puzzle, int, error) {
    offset := (page - 1) * size
    searchPattern := "%" + search + "%"

    // Build dynamic WHERE clause
    var whereClause string
    var countArgs, queryArgs []interface{}

    if role == "super_admin" {
        // Super admin sees ALL content (published and unpublished)
        whereClause = "WHERE p.judul ILIKE $1"
        countArgs = []interface{}{searchPattern}
        queryArgs = []interface{}{searchPattern, size, offset}
    } else {
        // Guru sees only their own content (published and unpublished)
        whereClause = "WHERE p.created_by = $1 AND p.judul ILIKE $2"
        countArgs = []interface{}{userId, searchPattern}
        queryArgs = []interface{}{userId, searchPattern, size, offset}
    }

    // Count query
    var total int
    countSQL := fmt.Sprintf(`SELECT COUNT(*) FROM puzzles p %s`, whereClause)
    err := r.DB.QueryRow(ctx, countSQL, countArgs...).Scan(&total)
    if err != nil {
        return nil, 0, err
    }

    // Main query — same SELECT columns as existing FindAll, but with dynamic WHERE
    // IMPORTANT: Remove the `is_published = true` filter!
    // Management endpoints should show ALL content (draft + published)
    limitOffset := fmt.Sprintf("LIMIT $%d OFFSET $%d", len(countArgs)+1, len(countArgs)+2)

    SQL := fmt.Sprintf(`SELECT p.puzzle_id, p.judul,
        COALESCE(ass_g.url, '') AS gambar, p.gambar_asset_id,
        COALESCE(ass_t.url, '') AS thumbnail, p.thumbnail_asset_id,
        p.kategori, p.xp_reward,
        JSON_BUILD_OBJECT(
            'user_id', u.user_id::text,
            'username', u.username
        ) AS user,
        p.created_at, p.is_published
    FROM puzzles p
    JOIN users u ON p.created_by = u.user_id
    LEFT JOIN assets ass_t ON p.thumbnail_asset_id = ass_t.asset_id
    LEFT JOIN assets ass_g ON p.gambar_asset_id = ass_g.asset_id
    %s
    ORDER BY p.created_at DESC
    %s`, whereClause, limitOffset)

    rows, err := r.DB.Query(ctx, SQL, queryArgs...)
    if err != nil {
        return nil, 0, err
    }

    puzzles, err := pgx.CollectRows(rows, pgx.RowToAddrOfStructByNameLax[entity.Puzzle])
    if err != nil {
        return nil, 0, err
    }
    return puzzles, total, nil
}
```

##### 1.1.3 Implement `FindByIdManage`

This method fetches a single content by ID but **enforces ownership for guru**:

```go
func (r *puzzleRepositoryImpl) FindByIdManage(ctx context.Context, puzzleId string, userId string, role string) (*entity.Puzzle, error) {
    // Build dynamic WHERE clause
    var whereClause string
    var args []interface{}

    if role == "super_admin" {
        whereClause = "WHERE p.puzzle_id = $1"
        args = []interface{}{puzzleId}
    } else {
        // Guru: must also match created_by
        whereClause = "WHERE p.puzzle_id = $1 AND p.created_by = $2"
        args = []interface{}{puzzleId, userId}
    }

    SQL := fmt.Sprintf(`SELECT p.puzzle_id, p.judul,
        COALESCE(ass_g.url, '') AS gambar, p.gambar_asset_id,
        COALESCE(ass_t.url, '') AS thumbnail, p.thumbnail_asset_id,
        p.kategori, p.xp_reward,
        JSON_BUILD_OBJECT(
            'user_id', u.user_id::text,
            'username', u.username
        ) AS user,
        p.created_at, p.is_published
    FROM puzzles p
    JOIN users u ON p.created_by = u.user_id
    LEFT JOIN assets ass_g ON p.gambar_asset_id = ass_g.asset_id
    LEFT JOIN assets ass_t ON p.thumbnail_asset_id = ass_t.asset_id
    %s`, whereClause)

    rows, err := r.DB.Query(ctx, SQL, args...)
    if err != nil {
        return nil, err
    }

    puzzle, err := pgx.CollectOneRow(rows, pgx.RowToAddrOfStructByNameLax[entity.Puzzle])
    if err != nil {
        return nil, err  // Will return "no rows" error if guru tries to access others' content
    }
    return puzzle, nil
}
```

> [!IMPORTANT]
> **For `FindByIdManage`**: If the query returns no rows, `pgx.CollectOneRow` will return an error. In the usecase layer, you should catch this and return `fiber.ErrNotFound` or `fiber.ErrForbidden` depending on whether the content exists at all or just doesn't belong to the guru.

```go
func (u *puzzleUseCaseImpl) UpdatePuzzleManage(ctx context.Context, puzzle *model.PuzzleRequest, puzzleId string, userId string, role string) (*model.PuzzleResponse, error) {
    err := u.Validator.Struct(puzzle)
    if err != nil {
        u.Log.Warnf("error when validate puzzle: %v", err)
        return nil, fiber.ErrBadRequest
    }

    // Ownership check: verify the puzzle belongs to this user (or user is super_admin)
    _, err = u.PuzzleRepository.FindByIdManage(ctx, puzzleId, userId, role)
    if err != nil {
        u.Log.Warnf("puzzle not found or access denied: %v", err)
        return nil, fiber.ErrForbidden
    }

    // Proceed with update (use existing Update method — it doesn't filter by created_by)
    UpdatedPuzzle := &entity.Puzzle{
        PuzzleId:         puzzleId,
        Judul:            puzzle.Judul,
        GambarAssetId:    puzzle.GambarAssetId,
        ThumbnailAssetId: puzzle.ThumbnailAssetId,
        Kategori:         puzzle.Kategori,
        XpReward:         puzzle.XpReward,
        IsPublished:      puzzle.IsPublished,
    }

    updatedPuzzle, err := u.PuzzleRepository.Update(ctx, UpdatedPuzzle)
    if err != nil {
        u.Log.Warnf("error when update puzzle: %v", err)
        return nil, fiber.ErrInternalServerError
    }

    // Mark assets as used (same as existing UpdatePuzzle)
    var assetIds []int
    if puzzle.GambarAssetId != nil {
        assetIds = append(assetIds, *puzzle.GambarAssetId)
    }
    if puzzle.ThumbnailAssetId != nil {
        assetIds = append(assetIds, *puzzle.ThumbnailAssetId)
    }
    if len(assetIds) > 0 {
        if err := u.AssetRepository.MarkAsUsed(ctx, assetIds); err != nil {
            u.Log.Warnf("failed to mark asset as used: %v", err)
        }
    }

    res := converter.ToPuzzleResponse(updatedPuzzle)
    return res, nil
}
```

```go
func (c *puzzleControllerImpl) GetAllPuzzleManage(ctx fiber.Ctx) error {
    page, _ := strconv.Atoi(ctx.Query("page", "1"))
    size, _ := strconv.Atoi(ctx.Query("size", "10"))
    search := ctx.Query("search", "")

    // Extract role from JWT claims
    claims := ctx.Locals("user").(*model.Claims)
    userId := ctx.Locals("userId").(string)
    role := claims.Role

    puzzles, total, err := c.PuzzleUseCase.GetAllPuzzleManage(ctx, page, size, search, userId, role)
    if err != nil {
        c.Log.Warnf("error when get all puzzle manage: %v", err)
        return err

    totalPages := (total + size - 1) / size

    res := model.WebResponse[[]*model.PuzzleResponse]{
        Data: puzzles,
        Paging: &model.PageMetaData{
            Page: page,
            Size: totalPages,
        },
    }
    return ctx.Status(fiber.StatusOK).JSON(res)
}
```

```go
func (c *puzzleControllerImpl) UpdatePuzzle(ctx fiber.Ctx) error {
    var puzzle model.PuzzleRequest
    if err := ctx.Bind().Body(&puzzle); err != nil {
        c.Log.Warnf("error when bind puzzle: %v", err)
        return fiber.NewError(fiber.StatusBadRequest, "bad request")
    }

    puzzleId := ctx.Params("id")
    claims := ctx.Locals("user").(*model.Claims)
    userId := ctx.Locals("userId").(string)
    role := claims.Role

    // Use Manage variant for ownership check
    updatedPuzzle, err := c.PuzzleUseCase.UpdatePuzzleManage(ctx, &puzzle, puzzleId, userId, role)
    if err != nil {
        c.Log.Warnf("error when update puzzle: %v", err)
        return err
    }

    res := model.WebResponse[*model.PuzzleResponse]{
        Data: updatedPuzzle,
    }
    return ctx.Status(fiber.StatusOK).JSON(res)
}
```

## ⚠️ Important Notes

> [!IMPORTANT]
> **Do NOT modify the existing public read methods** (`FindAll`, `GetAll`, `GetAllPuzzle`, etc.). These are used by the `allAuth` endpoints that serve published content to all roles including `member`. Create **new `*Manage` methods** alongside them.

> [!IMPORTANT]
> **Management endpoints remove the `is_published = true` filter.** When guru or super_admin view their content management dashboard, they need to see both **draft** and **published** content. The public read endpoints keep the published-only filter.

> [!WARNING]
> **Ownership check pattern**: For `guru` role, always verify `created_by = user_id` before allowing Update or Delete. Without this check, a guru could modify/delete another guru's content by guessing the ID. The `FindByIdManage` method enforces this at the SQL level.

> [!NOTE]
> **`Create` endpoints don't need role-based changes.** Both `guru` and `super_admin` already set `created_by` to their own `user_id` during creation. No ownership conflict is possible at creation time.

> [!NOTE]
> **Article (`artikel`) is intentionally excluded** from this issue since it follows the same pattern. If needed, apply the same changes to the article controller/usecase/repository in a follow-up issue.
