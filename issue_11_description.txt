# 🎓 Feature: Group CRUD Management & Member Invitation System (Guru Only)

## 📋 Summary

Implement a complete **Group CRUD** (Create, Read, Update, Delete) system that allows **Guru** (teachers) to create and manage groups, invite **Members** (students) via **email** and **QR code**, and monitor who has joined each group. Only users with the `guru` role should be able to manage groups. Only users with the `member` role can **join** a group.

---

## 🗄️ Database Context

### Table: `groups`
```sql
CREATE TABLE groups (
    group_id VARCHAR(191) NOT NULL,           -- UUID, primary key
    group_name VARCHAR(191) NULL,
    group_thumbnail_asset_id INTEGER NULL,     -- FK → assets(asset_id)
    created_by INTEGER NULL,                   -- FK → guru(guru_id)
    created_at TIMESTAMP(0) NULL,
    updated_at TIMESTAMP(0) NULL,

    CONSTRAINT groups_pkey PRIMARY KEY (group_id)
);
```

### Table: `group_members`
```sql
CREATE TABLE group_members (
    group_id VARCHAR(191) NOT NULL,           -- FK → groups(group_id) ON DELETE CASCADE
    member_id INTEGER NOT NULL,               -- FK → members(member_id) ON DELETE CASCADE
    tanggal_bergabung TIMESTAMP(0) NOT NULL DEFAULT CURRENT_TIMESTAMP,

    CONSTRAINT group_members_pkey PRIMARY KEY (group_id, member_id)
);
```

### Table: `guru`
```sql
CREATE TABLE guru (
    guru_id SERIAL NOT NULL,
    user_id INTEGER NULL,                     -- FK → users(user_id)
    sekolah_id INTEGER NULL,                  -- FK → sekolah(sekolah_id)
    nip VARCHAR(20) NULL,
    bidang_ajar VARCHAR(100) NULL,

    CONSTRAINT guru_pkey PRIMARY KEY (guru_id),
    CONSTRAINT guru_user_id_key UNIQUE (user_id),
    CONSTRAINT guru_nip_key UNIQUE (nip)
);
```

### Table: `members`
```sql
CREATE TABLE members (
    member_id SERIAL NOT NULL,
    user_id INTEGER NULL,                     -- FK → users(user_id)
    sekolah_id INTEGER NULL,
    nis VARCHAR(20) NULL,
    total_xp INTEGER NOT NULL DEFAULT 0,
    ...
    CONSTRAINT members_pkey PRIMARY KEY (member_id),
    CONSTRAINT members_user_id_key UNIQUE (user_id)
);
```

### Key Foreign Keys
```sql
ALTER TABLE groups ADD CONSTRAINT groups_created_by_fkey 
    FOREIGN KEY (created_by) REFERENCES guru(guru_id) ON DELETE SET NULL ON UPDATE CASCADE;

ALTER TABLE group_members ADD CONSTRAINT group_members_member_id_fkey 
    FOREIGN KEY (member_id) REFERENCES members(member_id) ON DELETE CASCADE ON UPDATE CASCADE;
```

## 🏗️ Architecture Overview

This project follows **Clean Architecture** with these layers:

```
Entity → Repository → UseCase → Controller → Route
```

| Layer | Directory | Purpose |
|---|---|---|
| Entity | `internal/entity/` | Database structs with `db:` tags |
| Model | `internal/model/` | Request/Response DTOs with `json:` and `validate:` tags |
| Converter | `internal/model/converter/` | Entity ↔ Model conversion functions |
| Repository | `internal/repository/` | Database queries using `pgx` |
| UseCase | `internal/usecase/` | Business logic + validation |
| Controller | `internal/delivery/http/` | HTTP handlers using `fiber.Ctx` |
| Route | `delivery/http/route/route.go` | Route registration + middleware |
| Middleware | `delivery/http/middleware/` | Auth + Role checking |
| DI/Bootstrap | `internal/config/app.go` | Dependency injection wiring |

**Tech Stack:**
- **Framework**: Go Fiber v3 (`github.com/gofiber/fiber/v3`)
- **Database**: PostgreSQL via `pgx/v5` pool (`pgxpool`)
- **Validation**: `go-playground/validator/v10`
- **Logging**: `logrus`
- **Auth**: JWT via `golang-jwt/jwt/v5`
- **UUID Gen**: `google/uuid`

### Step 4: Create Repository — `internal/repository/`

#### 4.1 Create `group_repository.go` [NEW]

> **File**: `internal/repository/group_repository.go`

Follow the same pattern as `article_repository.go` (interface + impl struct + constructor).

**Interface to define:**

```go
type GroupRepository interface {
    // Group CRUD
    CreateGroup(ctx context.Context, group *entity.Group) (*entity.Group, error)
    GetAllGroupsByGuru(ctx context.Context, guruId int, page int, size int, search string) ([]*entity.Group, int, error)
    GetGroupById(ctx context.Context, groupId string) (*entity.Group, error)
    UpdateGroup(ctx context.Context, group *entity.Group) (*entity.Group, error)
    DeleteGroup(ctx context.Context, groupId string) error

    // Group Members
    AddMember(ctx context.Context, groupId string, memberId int) error
    RemoveMember(ctx context.Context, groupId string, memberId int) error
    GetGroupMembers(ctx context.Context, groupId string) ([]*entity.GroupMember, error)
    IsMemberInGroup(ctx context.Context, groupId string, memberId int) (bool, error)

    // Helper
    GetGuruIdByUserId(ctx context.Context, userId string) (int, error)
    GetMemberIdByEmail(ctx context.Context, email string) (int, error)
    GetMemberIdByUserId(ctx context.Context, userId string) (int, error)
}
```

**Implementation details for each method:**

##### `CreateGroup`
1. Generate a new UUID for `group_id` using `github.com/google/uuid` — `uuid.New().String()`
2. `INSERT INTO groups (group_id, group_name, created_by, created_at, updated_at) VALUES ($1, $2, $3, NOW(), NOW()) RETURNING group_id`
3. After insert, call `GetGroupById` to return the full group with JOINs

##### `GetAllGroupsByGuru`
1. First run a `COUNT(*)` query filtered by `created_by = $guruId` and `group_name ILIKE $searchPattern` 
2. Then run the main query:
```sql
SELECT 
    g.group_id,
    g.group_name,
    COALESCE(a.url, '') AS group_thumbnail,
    g.group_thumbnail_asset_id,
    g.created_by,
    g.created_at,
    g.updated_at,
    JSON_BUILD_OBJECT(
        'GuruId', gu.guru_id,
        'NIP', gu.nip,
        'BidangAjar', gu.bidang_ajar,
        'Username', u.username
    ) AS guru,
    (SELECT COUNT(*) FROM group_members gm WHERE gm.group_id = g.group_id) AS member_count
FROM groups g
LEFT JOIN guru gu ON g.created_by = gu.guru_id
LEFT JOIN users u ON gu.user_id = u.user_id
LEFT JOIN assets a ON g.group_thumbnail_asset_id = a.asset_id
WHERE g.created_by = $1 AND g.group_name ILIKE $2
ORDER BY g.created_at DESC
LIMIT $3 OFFSET $4
```
3. Use `pgx.CollectRows(rows, pgx.RowToAddrOfStructByNameLax[entity.Group])` to scan

##### `GetGroupById`
Same query as above but filtered by `WHERE g.group_id = $1` instead, without pagination. Use `pgx.CollectExactlyOneRow`.

##### `UpdateGroup`
```sql
UPDATE groups SET group_name = $1, group_thumbnail_asset_id = $2, updated_at = NOW() 
WHERE group_id = $3 RETURNING group_id
```
After update, call `GetGroupById` to return the full group.

##### `DeleteGroup`
```sql
DELETE FROM groups WHERE group_id = $1
```
> The foreign key on `group_members` has `ON DELETE CASCADE`, so members will be automatically removed.

##### `AddMember`
```sql
INSERT INTO group_members (group_id, member_id, tanggal_bergabung) 
VALUES ($1, $2, NOW())
ON CONFLICT (group_id, member_id) DO NOTHING
```

##### `RemoveMember`
```sql
DELETE FROM group_members WHERE group_id = $1 AND member_id = $2
```

##### `GetGroupMembers`
```sql
SELECT 
    gm.group_id,
    gm.member_id,
    gm.tanggal_bergabung,
    u.username,
    u.email,
    m.nis,
    m.foto_profil
FROM group_members gm
JOIN members m ON gm.member_id = m.member_id
JOIN users u ON m.user_id = u.user_id
WHERE gm.group_id = $1
ORDER BY gm.tanggal_bergabung ASC
```

##### `IsMemberInGroup`
```sql
SELECT EXISTS(SELECT 1 FROM group_members WHERE group_id = $1 AND member_id = $2)
```

##### `GetGuruIdByUserId`
```sql
SELECT guru_id FROM guru WHERE user_id = $1
```
> This is needed because the JWT only contains `user_id`, but `groups.created_by` references `guru_id`.

##### `GetMemberIdByEmail`
```sql
SELECT m.member_id FROM members m JOIN users u ON m.user_id = u.user_id WHERE u.email = $1
```
> This is needed for email-based invitation — find the member_id from the invited user's email.

##### `GetMemberIdByUserId`
```sql
SELECT member_id FROM members WHERE user_id = $1
```
> This is needed when a member joins a group — extract their member_id from the JWT's user_id.

### Step 5: Create UseCase — `internal/usecase/`

#### 5.1 Create `group_usecase.go` [NEW]

> **File**: `internal/usecase/group_usecase.go`

Follow the same pattern as `article_usecase.go`.

**Interface to define:**

```go
type GroupUseCase interface {
    // CRUD
    CreateGroup(ctx context.Context, req *model.GroupCreateRequest, userId string) (*model.GroupResponse, error)
    GetAllGroups(ctx context.Context, userId string, page int, size int, search string) ([]*model.GroupResponse, int, error)
    GetGroupDetail(ctx context.Context, groupId string, userId string) (*model.GroupDetailResponse, error)
    UpdateGroup(ctx context.Context, groupId string, req *model.GroupUpdateRequest, userId string) (*model.GroupResponse, error)
    DeleteGroup(ctx context.Context, groupId string, userId string) error

    // Member Management
    InviteMembersByEmail(ctx context.Context, groupId string, req *model.GroupInviteEmailRequest, userId string) error
    GenerateInviteLink(ctx context.Context, groupId string, userId string) (*model.GroupInviteResponse, error)
    JoinGroup(ctx context.Context, req *model.GroupJoinRequest, userId string) error
    RemoveMember(ctx context.Context, groupId string, memberId int, userId string) error
    GetGroupMembers(ctx context.Context, groupId string, userId string) ([]model.GroupMemberResponse, error)
}
```

**Struct dependencies:**

```go
type groupUseCaseImpl struct {
    GroupRepository repository.GroupRepository
    AssetRepository repository.AssetRepository
    Log             *logrus.Logger
    Validator       *validator.Validate
    JWTSecret       []byte
}
```

**Implementation details:**

##### `CreateGroup`
1. Validate the request using `u.Validator.Struct(req)`
2. Get `guru_id` from `userId` via `u.GroupRepository.GetGuruIdByUserId(ctx, userId)`. If error → return `fiber.ErrForbidden` (user is not a guru)
3. Call `u.GroupRepository.CreateGroup(ctx, &entity.Group{GroupName: req.GroupName, CreatedBy: guruId})`
4. Convert result to `model.GroupResponse` using `converter.ToGroupResponse()`

##### `GetAllGroups`
1. Sanitize page/size (page >= 1, 1 <= size <= 50)
2. Get `guru_id` from `userId`
3. Call `u.GroupRepository.GetAllGroupsByGuru(ctx, guruId, page, size, search)`
4. Convert to response list

##### `GetGroupDetail`
1. Get `guru_id` from `userId`
2. Get group by ID, **verify** that `group.CreatedBy == guruId` (ownership check). If not → return `fiber.ErrForbidden`
3. Get group members via `GetGroupMembers`
4. Combine into `GroupDetailResponse` using converter

##### `UpdateGroup`
1. Validate request
2. Get `guru_id`, verify ownership (same as GetGroupDetail)
3. Call `UpdateGroup` repository
4. If `req.ThumbnailAssetId != nil`, mark asset as used via `AssetRepository.MarkAsUsed`
5. Convert and return

##### `DeleteGroup`
1. Get `guru_id`, verify ownership
2. Call `DeleteGroup` repository

##### `InviteMembersByEmail`
1. Validate request (emails array)
2. Get `guru_id`, verify ownership
3. For each email in `req.Emails`:
   a. Call `GetMemberIdByEmail(ctx, email)` to find the member
   b. If the member exists **and** their user role is `member`, call `AddMember(ctx, groupId, memberId)`
   c. If the member doesn't exist or is not a `member` role, skip (or collect errors)
4. *(Optional Phase 2)* Send an email notification to each invited email. For now, directly add them.

> **Important**: For the initial implementation, "invite via email" means: look up the user by email, verify they have the `member` role, and directly add them to the group. Actual email sending (SMTP) can be a separate follow-up task.

##### `GenerateInviteLink`
1. Get `guru_id`, verify ownership
2. Create a signed JWT invite token:
   ```go
   claims := jwt.MapClaims{
       "group_id": groupId,
       "guru_id":  guruId,
       "type":     "group_invite",
       "exp":      time.Now().Add(7 * 24 * time.Hour).Unix(), // 7 days expiry
   }
   token := jwt.NewWithClaims(jwt.SigningMethodHS256, claims)
   tokenString, _ := token.SignedString(u.JWTSecret)
   ```
3. Build the invite link: `{BASE_URL}/v1/groups/join?token={tokenString}`
4. For QR code: return the invite link as `QRCodeData` — the **frontend** will render the QR code from this URL string
5. Return `GroupInviteResponse` with `InviteToken`, `InviteLink`, `QRCodeData`, and `ExpiresAt`

##### `JoinGroup`
1. Validate request
2. Parse and verify the invite JWT token:
   ```go
   token, err := jwt.Parse(req.InviteToken, func(token *jwt.Token) (interface{}, error) {
       return u.JWTSecret, nil
   })
   ```
3. Extract `group_id` from the token claims
4. Verify the group still exists via `GetGroupById`
5. Get `member_id` from `userId` via `GetMemberIdByUserId(ctx, userId)`. If error → user is not a member → return `fiber.ErrForbidden`
6. Check if already in group via `IsMemberInGroup`. If already → return appropriate error
7. Call `AddMember(ctx, groupId, memberId)`

##### `RemoveMember`
1. Get `guru_id`, verify ownership
2. Call `RemoveMember` repository

##### `GetGroupMembers`
1. Get `guru_id`, verify ownership
2. Call `GetGroupMembers` repository
3. Convert to response

```go
func (c *groupControllerImpl) CreateGroup(ctx fiber.Ctx) error {
    req := new(model.GroupCreateRequest)
    if err := ctx.Bind().Body(req); err != nil {
        c.Log.Warnf("Invalid request body: %+v", err)
        return fiber.ErrBadRequest
    }
    userId := ctx.Locals("userId").(string)
    group, err := c.GroupUseCase.CreateGroup(ctx, req, userId)
    if err != nil {
        c.Log.Warnf("Failed create group: %+v", err)
        return err
    }
    return ctx.Status(fiber.StatusCreated).JSON(model.WebResponse[*model.GroupResponse]{Data: group})
}
```

##### `GetAllGroups`

- Call usecase, return with pagination metadata

##### `GetGroupDetail`
- Extract `groupId` from `ctx.Params("id")`
- Extract `userId` from Locals
- Call usecase

##### `UpdateGroup`
- Extract `groupId` from params, bind body, extract userId
- Call usecase

##### `DeleteGroup`
- Extract `groupId` from params, extract userId
- Call usecase, return success message

##### `InviteMembersByEmail`
- Extract `groupId` from params
- Bind body to `GroupInviteEmailRequest`
- Call usecase

##### `GenerateInviteLink`
- Extract `groupId` from params, extract userId
- Call usecase, return the invite response (token + QR data)

##### `JoinGroup`
- Bind body to `GroupJoinRequest`
- Extract `userId` from Locals
- Call usecase
- Return success message

##### `RemoveMember`
- Extract `groupId` from params
- Extract `member_id` from params (`:member_id`), convert to int
- Call usecase

##### `GetGroupMembers`
- Extract `groupId` from params, extract userId
- Call usecase, return member list

## ⚠️ Important Notes

> [!IMPORTANT]
> **Ownership Check**: Every group management endpoint (update, delete, invite, remove member) **must** verify that the requesting guru is the `created_by` owner of the group. Do not skip this.

> [!IMPORTANT]
> **Role Enforcement**: Group CRUD and member management = `guru` role only. Joining a group = `member` role only. Use `RoleMiddleware("guru")` and `RoleMiddleware("member")` respectively.

> [!WARNING]
> **guru_id vs user_id**: The JWT token contains `user_id`, but `groups.created_by` references `guru_id`. You **must** look up the `guru_id` from the `guru` table using `WHERE user_id = $1`. Do not use `user_id` directly as `created_by`.

> [!NOTE]
> **QR Code**: The backend returns the invite link URL as `qr_code_data`. The **frontend** is responsible for rendering this URL into a QR code image (using a library like `qrcode.js`). The backend does NOT generate the QR image itself.

> [!NOTE]
> **Email Sending**: For the initial implementation, "invite via email" directly adds the member to the group (if found by email). Actual SMTP email notification sending is a separate future task.
